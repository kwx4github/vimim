概述

一個文本文件，構成一個中文輸入法。插上聯網就可以雲輸入。
如果再加上一二個文本文件，無需聯網照樣敲打中文，隨心所欲。

程式文件用的是 vim script, 天生跨平台（Windows, Mac, Linux）
詞庫文件就是自己的輸入法（全拼或雙拼，五筆，英文，DIY）

所謂安裝，就是把文本文件扔到 vim 的 plugin 目錄之下。

【拼音★四角】同時插上，可以平行也可以交叉使用，無須切換。
【五筆★拼音】同時插上，可以平行使用，動態切換。

不加設置，不換模式，點熱鍵就可上漢字，是謂〖點石成金〗。
如果切換成中文輸入模式，也可以將就當前流行的輸入法習慣。

所謂卸載，就是刪除文本文件。

Unicode 內碼輸入

Unicode 使漢字成為一種理想的電腦語言，沒有國界。
用 VimIM 玩萬國碼的前提是 :set encoding=utf-8
［科普］ http://zh.wikipedia.org/wiki/Unicode

萬國碼輸入器

  • 【廣告】精巧細致的標准Unicode輸入器，獨此一家。
  • 【理想】書同文、文同碼、碼同鍵。
  • 【特點】與輸入法和詞庫都無關：以不變應萬變。
  • 【用途】內碼輸入是輸入非常用字符的有效途徑。
  • 【特點】一個字符對應一個代碼，武林高手可以中文盲打。
  • 【TIP】 UNICODE 內碼一次性列表　　:call CJK()

VimIM CLOUD 雲輸入 　雙拼

【VimIM 雲輸入】

  1. 〖全雲輸入〗每一個輸入都〖雲〗游

    • 特點：每打一個字母都要向服務器請求一次。
    • 手動開啟：可設置 :let g:vimim_cloud_sogou=1
    • 自動開啟：
      • Linux 　如果vim plugin目錄下沒有詞庫
      • Windows　如果wget.exe置於vim plugin目錄之下

  2. 〖可控性雲輸入〗每N個輸入就〖雲〗游

    • 特點：每打N個字母才向服務器請求一次。
    • 設置：:let g:vimim_cloud_sogou=N
    • （長的）詩詞曲賦、新聞八卦、完整的句子拜托給〖雲〗。
    • （短的）私人信息、英文、詞組、單字留給VimIM自己處理。
    • 長短的定義沒有統一的國際標准。玩家自己定標准。
    • 例如：如果選項設置為10:　即每敲10個字母以上就丟給〖雲〗。
    • 結果：本地詞庫和搜狗雲端詞庫智能切換。

  3. 支持〖斷字符〗　解決含糊的辦法是在含糊的地方貼上標簽：

    • 錯：xianmeimeidepiaosuifengpiaoyang　=>　先美美地飄隨風飄揚
    • 對：xi'anmeimeidepi'aosuifengpiaoyang　=>　 西安妹妹的皮襖隨風飄揚

  4. 支持中文輸入模式〖經典動態〗輸入風格 （缺省）

VimIM 可使多種輸入方式和諧並存，互不沖突而又相互補充。
VimIM 充分發揮個人的知識潛能，達到“人盡其能、物盡其用”的境界。
  • 【用途】單碼（音碼或形碼）為主，雙碼（音碼加形碼）為輔。
  • 【單碼特點】二個單碼可混合使用，不用切換。

  • 【結論】最好的輸入法是自己的輸入法。

【主要問題】
經典音碼：一音多字，一字多音。
流行形碼：難學難記，萬碼奔騰。
現代混碼：難記難學，各自為戰。

　　　　第三步：按照自己定的規矩：想怎麼打，就怎麼打！
【答案】模糊 + 模糊 = 精確

VimIM 是什麼意思?
答:
VimIM 的本義應該是 Vim Input Method —— Vim 輸入法。
因為本文是中文版本，VimIM 就順勢譯成 Vim 中文輸入法。

VimIM 的作者是誰?
答:
作者的網名是 vimim 　 :)
問:
VimIM 與 latex-suite plugin 好像有點沖突?
答:
有玩家反應。Let's wait for solution from those players who
are interested in making both plugins work in harmony.
問:
VimIM 與 Vim abbreviation 好像有點沖突?
答:
Yes, but ...
如果不激活 VimIM, Vim 的abbreviation永遠不會“被和諧”。

問題好像隻出現在用〖點石成金〗上字之后，接著馬上敲空格鍵，
而且敲空格鍵的目的是激活abbreviation. 不過這時候，再次用
空格鍵時，空格鍵又可以激活abbreviation了。

原因：OneKey需要賦予空格鍵一點智能，以便於上字。
問:
怎樣減少 copy & paste (復制粘貼)重復操作?

VimIM 設計原則　　 Design Goals

VIM:不被和諧!
memory:不超過詞庫尺寸
speed:不低於最高要求
encoding:不受限制
options:不強行要求設置
internet:不聯網照樣敲中文
datafile:無詞庫可以雲輸入

VimIM 設計思路:

  • 用文本文件打造中文輸入法：文本文件，天下無敵。
  • 一個文本文件就足以聯網 CLOUD 雲輸入
  • 兩個文本文件就是完整的中文輸入法。
  • 詞庫文件完全歸用戶管理，可以直接用Vim編輯。
  • 源程式不用編譯，修改以后立竿見影。

［科普］如何讓Vim支持中文？

  1. Vim 支持中文編碼的基礎

    • Vim 支持中文編碼的基礎是兩個特性：+multi_byte 和 +iconv
    • 官方下載網址： http://www.vim.org/download.php
    • 綠色移動版： http://portablegvim.sourceforge.net/downloads.html

  2. 影響 Vim 中文編碼的設置項

    • encoding：vim 的內部使用編碼
    • fileencodings：Vim 在打開文件時會根據其選項來識別文件編碼
    • fileencoding：Vim 在保存新建文件時會根據其設置編碼來保存
    • guifontwide：Gvim 要求等寬字體，其指定字寬是 guifont 的兩倍

  3. Vim 具體應用環境的設置：微軟視窗（Windows） vimrc 樣例

    • set encoding=utf-8
    • set fileencodings=ucs-bom,utf8,chinese,taiwan,ansi
    • set guifont=Courier_New:h12:w7
    • set guifontwide=NSimSun-18030,NSimSun
    • set ambiwidth=double

  4. 如何用 Vim 插入非 ASCII 字符？

    如果既不依賴外挂程式，又不干擾 Vim 正常操作，
    Vim 可以選擇如下三種方式，插入不能用普通鍵盤輸入的字符：
    1. 用 i_CTRL-V 插入十進制編碼
    2. 用 i_CTRL-K 插入二合字母
    3. 用 i_CTRL-6 〖點石成金〗 （VimIM 推陳出新）
